Corrections

Every error we make, on the record.

A civic-transparency project earns trust by being audited, not by claiming to be infallible. This page is the public log of every correction we have issued.

The promise

  • Every reported error gets reviewed within 7 days.
  • Confirmed errors are corrected on the source data and recorded here, with a date and a one-line description of what changed.
  • We do not silently rewrite history. The git commit history preserves every change to every record; this page is the human-readable index of the corrections that mattered.
  • If a correction involves a person or organisation, they get a right of reply — see the code of conduct.

How to report an error

Three ways, pick whichever is easiest for you:

  1. Email ravikntsh@gmail.com with the project ID, the field in question, and your source for the correct value.
  2. GitHub issue at github.com/Raviknight/nirman-darpan/issues — preferred if you can include a citation URL.
  3. Sign in and comment on the project — flags also reach us this way, slower.

Reports without a source citation will be reviewed but probably not actioned — we don't change a recorded number on someone's word alone. With a source, turnaround is usually 1–3 days.

Right of reply

If you are an official, contractor, or organisation named in any project record or accountability entry and want to respond, write to ravikntsh@gmail.com. Your response will be published next to the entry it concerns, without editing for tone. We may add a short editorial note giving context.
